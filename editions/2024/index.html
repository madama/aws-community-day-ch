    Raphael Manke
    Building Reliable Serverless Applications with AWS CDK and Testing
    AWS CDK is an IaC (Infrastructure as Code) Framework that allows developers to easily create and manage serverless applications by combining various AWS resources such as Lambda, API Gateway, and DynamoDB tables with just a few lines of code. While CDK greatly simplifies the process of building and deploying these types of applications, it is still important to verify that they are functioning as expected. In this talk, we will delve into the different ways of testing CDK apps to ensure their reliability, using a real-world example to illustrate the concepts discussed. This will include a discussion of unit testing Lambda functions, snapshot testing, and fine-grained assertion testing of CDK stacks. We will also explore the importance of integration testing in gaining confidence in your application, and demonstrate how to split an application into smaller, more manageable components that can be deployed and tested individually. By the end of this talk, attendees will have gained a thorough understanding of the various testing techniques available when working with CDK, and will be equipped with the knowledge and skills to confidently ensure the reliability of their serverless applications through effective testing practices.
    Raphael Manke

    Read more

    Jochen Zehnder
    Deploying Secure Landing Zones in highly regulated sectors

    Have you ever wondered how cloud infrastructure really works, especially in places where the rules are strict, like financial services and banking? Well, Landing Zones are crucial to getting things right, but boy, can they vary—across different platforms and even within AWS itself!

    Join us as we break down the basics of AWS Landing Zones with a unique look at how they're put together for regulated industries. We'll walk you through some crucial design decisions and what they mean for keeping your AWS accounts secured and running smoothly. It's not just about setting things up but ensuring everything stays as it should—no sneaky misconfigurations allowed!

    We'll also discuss how to run your applications using an Amazon EKS cluster as the platform foundation. How many clusters should you manage? What about services that don't fit inside the cluster? And who is in charge of setting these up and keeping them going?

    To simplify your life, you can achieve all this using Infrastructure as Code (IaC). You'll learn some handy tips and tricks on implementing IaC and the best patterns to use for different parts of your setup.

    So, whether you're looking to tighten security or streamline operations, this talk will help you navigate the complexities of deploying Landing Zones in sectors where compliance isn't just a suggestion—it's a must.

    Thorsten Höger
    From Zero to Hero - Frictionless CI/CD pipelines in projen
    In this presentation, I will explore the innovative concept behind projen-pipelines, a tool designed to revolutionize how we approach continuous delivery (CD) pipelines for AWS CDK applications. The talk will focus less on the technical aspects of the library itself and more on the transformative idea it represents and the ambitious goals it aims to achieve. Projen-pipelines is not just a tool; it's a vision for a more efficient, streamlined, and user-friendly approach to managing complex, multi-environment, and multi-account AWS CDK applications. It embodies simplifying infrastructure management by abstracting away the complexities traditionally associated with defining and managing AWS CDK applications. We will discuss the goals of projen-pipelines, which include reducing the amount of code needed to handle complex deployment scenarios, enhancing efficiency in infrastructure management, and making the process more straightforward for developers of all experience levels. The talk will also touch on the future of projen-pipelines, including its potential to foster a more collaborative and inclusive open-source community. We will discuss how the tool encourages user contributions, fostering a community-driven approach to continuous improvement and innovation. Join me as we delve into projen-pipelines' transformative potential and explore how it aims to redefine our approach to AWS CI/CD pipelines.

    Serverless ❤️ Green Software - Making Batch Jobs More Sustainable

    Green IT is gaining increasing importance, and as an industry, we must actively contribute to addressing climate change by ensuring that our software doesn't impose unnecessary environmental burdens.

    In this presentation, you get introduced to the concept of Green Software Development and we dive into the idea of Demand Shifting, specifically Time Shifting. It will be demonstrated how we can put this concept into practice. Through efficient scheduling, you can reduce the CO2 emissions of your (batch) jobs.     A key takeaway from this talk is that sustainability in IT isn't just an idea; it can be seamlessly integrated into our daily routines with minimal effort. Moreover, the methods I'll showcase are applicable to other cloud providers and environments as well.

    Michal Salanci
    More Network Security: Forward Proxy on AWS Network Firewall & Fargate

    Our customer asked as for powerfull, and scalable forward proxy, so we mixed AWS Network Firewall, with Squid Proxy, Fluentbit, Dnsmasq and Telegraf containers powered by Amazon Fargate to craft this juicy serverless smoothie.

    🌐 AWS Network Firewall is main indigence as a transparent proxy. We combined stateful and stateless NFW rules for powerful, scalable and AWS managed transparent forward proxy. NFW rules are whitelisting the IPs and URLs our customer wanted, and blocking everyting else.

    There is always a demand for good old classic stuff, that's why 🦑 Squid Proxy (open source) is backing up AWS Network Firewall and serves as explicit proxy, as there was a demand for it. We put this tool into a ECS container for easier… everything. Fluentbit log forwarder runs as sidecar container in the same ECS task, for forwarding the logs into AWS CloudWatch.

    Using the forward proxy, our customer started to face the let's encrypt issue therefore 🔧 Dnsmasq (open source) running in ECS container was included to solve that. It acts as a DNS forwarder and resolves the DNS challenges coming with let's encrypt, when trying to obtain SSL/TLS certificates.

    Every good tool needs to me montitored, so 📈 Telegraf (open source) is here to help us with monitoring the performance of both transparent and explicit proxies in a near real time. It runs in ECS container in a "service" VPC and constantly monitoring the performance. AWS CloudWatch metrics are sent to Grafana for better visualization, and in case of any issue AWS CloudWatch alarm calls for action.

    All those containers would be difficult to manage, so it's all managed by 🧙‍♂️ Amazon Fargate a serverless solution for ECS containers to do all the magic we are lazy to do.

    Last but not least - putting all the proxy components into a single VPC, gives us better log visibility, higher level of security and reduced cost.

    Raphael Manke
    You don't need a Lambda for this. Refactor your serverless app.
    Are you interested in learning about other options for building and deploying applications using serverless architecture? In this talk, we will explore alternatives to AWS Lambda, such as AWS EventBridge, Step Functions, and EventBridge Pipes. We will discuss the benefits and drawbacks of each option, as well as some real-world use cases. Additionally, we will examine how to reduce the number of Lambdas needed in your application by replacing them with configuration rather than code. To demonstrate these concepts in action, we will also build a sample application using the AWS CDK framework. The AWS CDK is an open-source software development framework for defining cloud infrastructure in code and provisioning it through AWS CloudFormation. By using the CDK, we can leverage the power of familiar programming languages and modern software development practices to build and deploy cloud-native applications. By the end of this talk, attendees will have a better understanding of the various serverless technologies available and how to choose the best option for their needs. They will also learn how to leverage these technologies and the AWS CDK to build and deploy cloud-native applications that are scalable, reliable, and cost-effective.

    Waldemar Hummer
    LocalStack: Turbocharging feedback loops for developing AWS cloud apps

    This talk provides a hands-on introduction to LocalStack - the leading platform for developing and testing cloud applications entirely on your local machine.

    LocalStack provides a high-fidelity emulator for some 90+ AWS services, running in a local Docker container. The popular open source project (52k+ Github stars, 230+ million downloads) is today considered a "must-have" in the toolbox of AWS cloud developers around the globe.

    This interactive session covers live coding to showcase common use cases, different settings for local debugging of Lambdas and containerized apps (e.g., ECS), as well as advanced features that can radically improve team collaboration.

    We will also glance over the large ecosystem of tools that LocalStack natively integrates with (from IaC frameworks like Terraform or CDK, to application toolkits like Serverless, SAM, or Copilot) as well as some of the enterprise features in the platform.

    The session will close with a short peek into the architectural internals of LocalStack, as well as an outlook on some of the exciting features on the roadmap.
